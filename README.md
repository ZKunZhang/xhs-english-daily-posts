# 小红书英语内容工作区

这个仓库用于管理英语学习类小红书内容，覆盖账号定位、选题、写稿、制图、待发布、已发布归档和数据复盘。

## 账号定位

`生活场景英语易混点整理号`

每天帮用户分清一组“看着会，其实容易用错”的高频英语表达。内容先讲生活场景或具体后果，再用一句话讲清规则。

当前策略详见：

- [账号定位](07_账号定位/账号定位.md)
- [06-22 数据结论](07_账号定位/数据结论_20260622.md)

## 当前执行入口

1. 从 `02_内容生产/待发布/主线/` 查看当日可发布稿件。
2. 发布前搜索最新笔记明细表和已发布归档，确认知识点没有发过。
3. 按完整发布稿生成图文并发布。
4. 发布后将稿件移入 `04_内容归档/已发布/`。
5. 新数据导出后，按 `05_数据复盘/复盘规则.md` 复盘。

## 内容生产规则

### 选题

优先选择：

- 两边都成立的高频熟词或短语。
- 中文里容易混成一个意思，英文中有明确边界。
- 能用生活动作、位置或错误后果可视化展示。
- 能用一句话讲清区别。

当前内容比例：

- `70%`：低门槛熟词强对比。
- `20%`：高频结构辨析。
- `10%`：合集、答疑或实验内容。

### 标题和封面

- 首选“微小差异 + 具体后果”，例如“多一个 the，人就进车里了”。
- 没有更强场景时，再用 `A 和 B 原来不一样`。
- 不用“高考易错”、“每日英语”等栏目词抢占主标题。
- 封面只保留主钩子和一行短副标题，不堆规则和例句。

### 图文结构

1. 第 1 页：生活场景或错误后果封面。
2. 第 2 页：两个真实例句对比。
3. 第 3 页：一句话规则。
4. 第 4 页：两道同场景自测题。
5. 第 5 页：答案、简短理由和记忆口诀。

正文保留“场景/题目 → 答案 → 规则 → 例句 → 自测 → 互动”节奏。涉及时态、动作先后或 `to do/doing` 时，补充时间线解释。

## 已发布归档

`04_内容归档/已发布/` 包含：

- 根目录下的日期文件夹：正式发布记录，优先保留完整发布稿和图片提示词。
- `00_早期文本记录/`：只有 `内容.txt` 的早期发布记录。

已发布内容不应继续留在待发布目录。去重时先看最新笔记明细表，再搜索已发布归档和历史明细表。

## 数据复盘

`05_数据复盘/` 只保留：

- 最新一组观看数据和笔记明细 Excel，作为下次增量对比基线。
- [复盘规则](05_数据复盘/复盘规则.md)。

复盘节奏：

- 每天记录。
- 每 3 天小复盘，可调选题顺序和封面。
- 每 7 天正式复盘，才调整内容比例和策略。

新数据分析完成后，删除旧 Excel，只保留最新一组。过期的“今日发布建议”和临时阶段复盘不长期保留。

## 清理规则

- 项目说明只维护本 README，不在子目录重复建立 README。
- 系统文件、空目录、临时文件和已失效的临时分析不保留。
- 历史已发布稿只做去重和写法参考，不直接回流到当前排期。
- 不删除尚未确认用途的稿件；日期和归档状态冲突时，先核对发布记录。
