# 强对比辨析系列｜低门槛强对比版

日期范围：2026-06-10 到 2026-06-16

这个系列不继续铺泛语法题，优先做“看一眼就懂、不同学段都能看”的强对比辨析。

## 2026-05-18 数据复盘更新

最新导出的 `数据分析/笔记列表明细表.xlsx` 显示，`in time 和 on time 原来不一样` 已经成为当前最高观看笔记：

- `in time 和 on time 原来不一样`：曝光 `1954`，观看 `428`，封面点击率 `13.8%`，点赞 `25`，收藏 `25`，涨粉 `4`
- `job 和 work 原来不一样`：曝光 `2437`，观看 `391`，封面点击率 `13.2%`，点赞 `11`，收藏 `9`
- `by mistake 这个短语一定要记住`：曝光 `1529`，观看 `202`，封面点击率 `11.9%`，点赞 `16`，收藏 `14`，涨粉 `4`
- 2026-05-17 发布的两篇还处在早期数据阶段，表内出现 `曝光=0` 但有观看量，暂不直接判定内容失败

新的判断有三层：

1. `in time / on time` 强在内容门槛低。小学高年级、初中、高中、成人学习者都能理解，不需要先懂时态、非谓语或复杂语法术语。
2. `by mistake` 观看不算最高，但涨粉和最高观看笔记持平；收藏/观看约 `6.9%`，涨粉/观看约 `2.0%`，都高于 `job / work`。说明“这个短语一定要记住”这类标题更容易让用户觉得账号值得关注。它比单纯的 `A 和 B 原来不一样` 更直接给出保存和关注理由。
3. 补回的 `outputs/基础语法系列/day_05_2026-05-15_by_mistake/完整发布稿.md` 显示，`by mistake` 的内容结构是“填空题 -> 答案 -> 固定搭配 -> 例句 -> 口语替换”，不属于纯 A/B 辨析。后续不要为了统一系列形式，把这类选题强行改成两个表达对比。

## 2026-05-19 数据复盘更新

新增数据继续支持“高频混淆词”和“别再写错”两个方向：

- `by mistake 这个短语一定要记住`：目前最强转粉模板。它不是最高曝光模板，但能直接给用户“这条值得记住、这个账号值得关注”的理由，后续固定短语类继续优先测试 `X 这个短语一定要记住`。
- `home 和 house 原来不一样`：曝光 `1549`，观看 `216`，封面点击率 `12.7%`，涨粉 `2`。说明“高频混淆词”方向仍然有效，尤其是中文一个词对应英文两个常见词的题。
- `discuss 后面别再加 about 了`：曝光 `994`，观看 `138`，封面点击率 `13.7%`。说明“别再写错”标题很适合这个账号，适合用在介词搭配、动词搭配、中文直译误用类选题。

新的执行判断：

1. `A 和 B 原来不一样` 继续用于高频混淆词，重点追点击率和稳定曝光。
2. `X 这个短语一定要记住` 继续用于固定短语、搭配和口语表达，重点追涨粉率。
3. `X 后面别再加/用 Y 了`、`X 别再写错` 继续用于明确错误纠正，重点追封面点击率。

## 低门槛选题规则

主表已按这个方向更新，优先级从高到低：

- 优先：`in time / on time`、`job / work`、`fit / suit`、`home / house`、`travel / trip` 这类日常词和生活场景
- 优先：中文一个词、英文分两个表达，例如“家”“工作”“旅行”“准时/及时”
- 优先：不需要语法背景，第一眼能看懂题目，5 页图内能讲完
- 优先：标题能直接写成 `A 和 B 原来不一样`
- 优先：固定短语、介词搭配、中文直译误用类，标题能直接写成 `X 这个短语一定要记住`
- 优先：错误搭配纠错类，标题能直接写成 `X 后面别再加/用 Y 了`、`X 后面别再用错`、`X 别再写错` 或 `X 前面的介词不是 Y`
- 后置：`remember doing / remember to do`、`will be doing` 这类需要时间逻辑或语法术语的题
- 后置：`economic/economical`、`principal/principle`、`compliment/complement` 这类高阶词汇辨析

## 统一模板

这套系列继续采用强对比辨析模板：

- A/B 真对比标题优先级：`A 和 B 原来不一样` > `A 和 B，真的不是随便换` > `别再把 A 和 B 当一个词`
- 固定短语/搭配纠错标题优先级：`X 这个短语一定要记住` > `X 后面别再加/用 Y 了` > `X 别再写错` > `X 前面的介词不是 Y` > `别再把 X 写错`
- 标题选择原则：能让用户立刻知道“这条值得收藏、这个账号值得关注”的标题优先；不要为了统一形式强行套成 A/B
- 内容结构：`题目页 -> 答案页 -> 规则页 -> 例句页 -> 避错页`
- 选题原则：优先做中文母语者高频混淆、口语或写作里真会用到、低年级也能看懂的表达
- 图片风格：图 1 封面使用真实纸质单页、正面俯拍、浅米白纸面、少量红笔/绿勾；第 2-5 页保留轻笔记感和少量高亮
- 封面安全区：关键信息集中在中间 70%，左下角和右下角留白，不放关键字
- 封面文字：只保留顶部小字、中间超大主标题、下方一行短副标题
- 避免风格：不要继续使用密集便签、贴纸堆叠、复杂插画、硬表格、PPT 感封面

后续新增内容不再按日期创建文件夹，统一输出为 `outputs/` 下的 Markdown 表格。表格只使用 8 列：

```text
发布标题｜标题备选1｜标题备选2｜标题备选3｜图片提示词｜发布内容｜标签｜评论
```

## 发布判断口径

- 新发 2 小时内，如果曝光为 0 或 CTR 为 0，不单独判断内容失败，优先等 24 小时数据。
- 强对比辨析系列重点看：封面点击率、收藏率、涨粉、评论质量。
- 活动笔记看承接能力，不看曝光；重点看活动入口后的观看、互动、关注、私信或评论承接。
- 自然笔记看曝光和点击率；先判断选题和封面能不能拿到自然推荐。
- 转粉笔记看涨粉率；优先用涨粉/观看判断标题和内容是否真的建立关注理由。
- 单条失败不停止系列；连续 3 条 24 小时数据都低于基础语法系列中位数，再调整选题或封面。
